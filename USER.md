# 项目人类赞助者（Sponsor）档案

## 技术背景

- 熟练使用 Claude Code，依赖 AI 辅助完成工程任务
- 代码能力：入门至中级，可读懂简单逻辑，不擅长深入调试
- 日志解读：看不懂复杂日志，需要 AI 翻译成人话再决策

## 决策风格

- 擅长选择题，不擅长开放性问题
- **每个决策点必须给出 Top 3 方案**，附简短利弊说明，由我选一个
- 不要抛出"你想怎么做"的开放问题，直接给选项

## 项目目标

- 对给出的问题进行调研
- 期待产出业界中比较先进的解决方案或答案
- 重视方案的前瞻性与实用性并重

## 工作风格

- 遵循 zero-review 默认原则，无需另行指定
- AI 自主判断任务类型并走对应 paradigm（new / enhance / fix / add）
- 不需要额外征求风格偏好，直接执行

## 沟通规范

- **所有产出必须附中文摘要**（技术文档、分析报告、代码变更说明均适用）
- 决策点统一格式：

```
## 决策：<问题>

| # | 方案 | 优点 | 缺点 |
|---|------|------|------|
| 1 | ...  | ...  | ...  |
| 2 | ...  | ...  | ...  |
| 3 | ...  | ...  | ...  |

> 推荐：方案 X（一句话理由）
```

- 日志、报错等复杂输出需先翻译成业务语言再呈现
