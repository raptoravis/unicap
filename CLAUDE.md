## Repository Purpose

This is a **documentation-only repository** — a methodology handbook for scaling AI-assisted development from 1 agent to 3–6 concurrent agents. There is no source code, build system, or test suite. The primary artifact is `docs/methodology.md`.

## Sponsor Profile

Always read `USER.md` before producing output. Key constraints:

- **Decisions**: Present exactly **Top 3 options** with a comparison table; never ask open-ended questions.
- **Language**: All summaries and decision points must be in **Chinese**.
- **Complexity**: Translate logs/errors into business language before showing them.
- **Workflow**: Follow zero-review paradigm by default; auto-classify task type.

## Methodology Summary (docs/methodology.md)

The handbook is organized around three keys and four parallelization modes:

**Three Keys**
1. **需求对齐 (Requirement Alignment)** — Agent exhaustively questions ambiguities, presents Top 3 options; human only makes binary choices. See §2.2 for the reusable prompt template.
2. **TPDD (Test Plan-Driven Development)** — Test plan is generated by Agent A, reviewed independently (separate session), then locked before Agent B writes any code. See §3 for the flow.
3. **工程纪律 Skill 化 (Engineering Discipline as Skills)** — Coding standards (module depth, info hiding, naming, error policy, PR summary) are encoded as Skills in `.claude/skills/` and loaded at every Agent startup. See §4 for the required Skill list.

**Four Parallelization Modes** (ascending difficulty)
1. Cross-project — different repos, different Agents, zero coordination cost.
2. Same project, different modules — use `git worktree`; requires module file overlap < 20%.
3. Same module, different transaction types — implementation Agent / test Agent / bug-fix Agent in parallel worktrees.
4. Intra-task sub-module parallelization — Opus 4.7 capable but unstable; attempt only after 4+ weeks on modes 1–3.

**Event contract** (`events.*`) is the single shared coupling point across modules; it must be locked first (serially) and treated as frozen thereafter.

## Skills Convention

Project-level Skills belong in `.claude/skills/`. The full recommended set is listed in Appendix A of `docs/methodology.md`:

```
naming.md · module-depth.md · info-hiding.md · error-policy.md
pr-summary.md · invariants.md · replay-suite.md · events-frozen.md · tpdd.md
```

Keep each Skill under 200 characters of rules; lead with examples over prose.

## PR and Merge Rules

- Single PR: < 500 lines diff, < 7 files.
- Merge cadence: 2–4 times per day; avoid Friday merges.
- Every PR must include a structured summary matching the template in §7.1 of the methodology (reviewable in < 90 seconds).
- 100% manual spot-check required for: new cross-module interfaces, schema changes, new dependencies, deleted test cases, single-file diffs > 300 lines.
